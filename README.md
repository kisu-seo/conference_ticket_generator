## My process

### Built with

- **Vanilla JavaScript** — Implements drag-and-drop / click-to-upload avatar handling with file type and size validation, live form validation for name, email, and GitHub username with real-time error clearing, dynamic ticket generation with a randomly generated ticket number, and a viewport-based scaling routine that keeps the fixed-size ticket card responsive.
- **Tailwind CSS 4** — Handles all styling via utility classes with a CSS-first `@theme` design token system (colors, border radius, spacing) and custom `@utility` classes for typography presets and gradients, plus responsive breakpoints for mobile, tablet, and desktop layouts.
- **Vite 5** — Serves as the frontend build tool, offering a fast development server (HMR) and an optimized production build via the `@tailwindcss/vite` plugin.
- **Semantic HTML5 & Accessibility (A11y)** — Built with semantic tags (`<main>`, `<section>`, `<header>`), ARIA attributes (`aria-label`, `aria-describedby`, `aria-invalid`, `role="button"`, `role="img"`, `role="alert"`), and keyboard-accessible interactive controls throughout.
